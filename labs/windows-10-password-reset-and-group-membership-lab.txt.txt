# Windows 10 Password Reset and Group Membership Lab

## Goal
Practice common Tier 1 Help Desk tasks involving local user account creation, password reset, login verification, and group membership changes.

## Environment
- Host OS: Windows 11
- Virtualization: VirtualBox
- Guest VM: Win10-Lab-01
- Guest OS: Windows 10 Pro
- Working admin-capable account: LabUser
- Test account: ResetTest1

## Steps Performed
1. Started the Windows 10 VM `Win10-Lab-01` in VirtualBox and signed in as `LabUser`.
2. Opened an elevated Command Prompt.
3. Ran `net user` to review existing local accounts before making changes.
4. Created a new local test account named `ResetTest1` with:
   `net user ResetTest1 TempPass123! /add`
5. Verified the new account with:
   `net user ResetTest1`
6. Reset the password for `ResetTest1` with:
   `net user ResetTest1 NewPass123!`
7. Signed out of `LabUser` and successfully logged in as `ResetTest1` using the new password.
8. Verified the logged-in account with:
   `whoami`
9. Checked the account details and group membership with:
   `net user ResetTest1`
10. Signed out and confirmed the old password `TempPass123!` no longer worked.
11. Signed back in as `LabUser`.
12. Added `ResetTest1` to the local Administrators group with:
    `net localgroup Administrators ResetTest1 /add`
13. Verified that `ResetTest1` was now a member of `Administrators`.
14. Signed in again as `ResetTest1`.
15. Opened Command Prompt as administrator to confirm the account could elevate successfully.
16. Verified account identity and membership again with:
    `whoami`
    `net user ResetTest1`
17. Signed back in as `LabUser`.
18. Deleted the temporary test account with:
    `net user ResetTest1 /delete`
19. Ran `net user` again to confirm `ResetTest1` was removed.

## Commands Used
net user
net user ResetTest1
net user ResetTest1 TempPass123! /add
net user ResetTest1 NewPass123!
whoami
net localgroup Administrators ResetTest1 /add
net user ResetTest1 /delete

## What Worked
- Successfully created the local test user `ResetTest1`.
- Successfully verified the account existed and was active.
- Successfully reset the password as an administrator without needing the old password.
- Successfully logged in with the new password.
- Successfully confirmed the old password no longer worked.
- Successfully added `ResetTest1` to the local `Administrators` group.
- Successfully verified that the account could elevate with "Run as administrator."
- Successfully deleted the test account and confirmed it was removed.

## Problems I Hit
- I noticed temporary mouse lag in the VirtualBox VM during the lab.

## How I Resolved Them
- The mouse lag stopped during the session and did not require additional changes.

## What I Learned
- It is important to verify the real outcome of an account change by testing sign-in, not just trusting that a command succeeded.
- Group membership changes should also be verified after they are made.
- Local administrator access can be confirmed both by checking group membership and by testing elevation with "Run as administrator."
- Temporary test accounts are useful for practicing support tasks safely.
